Program 1

Artificial Neural Network (Backpropagation) for Regression

Section 1 — Aim

To implement a feed-forward Artificial Neural Network with one hidden layer using backpropagation to predict student marks from study hours and sleep hours.

Section 2 — Problem Statement

Build a 2→3→1 ANN from scratch using NumPy. Normalize inputs and outputs, train with sigmoid activation and gradient descent for 7000 epochs, and minimize Mean Squared Error between predicted and actual marks.

Section 3 — Theory

ML Concept

Artificial Neural Networks (ANN) mimic biological neurons. This is a supervised regression problem — we learn a mapping from input features to a continuous output.

Why Backpropagation?

Backpropagation efficiently computes gradients of the loss with respect to weights using the chain rule, enabling gradient descent to update weights layer by layer.

Key Formulas

Sigmoid: σ(x) = 1 / (1 + e−x)
Sigmoid derivative: σ′(x) = σ(x) · (1 − σ(x))
Forward pass (hidden): h = σ(X · Wh + bh)
Forward pass (output): ŷ = σ(h · Wout + bout)
Loss (MSE): L = mean((y − ŷ)²)
Weight update: W ← W + α · gradient · learning_rate

Advantages

  • Can model non-linear relationships
  • Universal function approximator with enough neurons
  • From-scratch implementation builds deep understanding

Limitations

  • Slow convergence; sensitive to learning rate
  • Vanishing gradient with sigmoid in deep networks
  • Small dataset (3 samples) — for demonstration only
  • No validation split; may overfit on tiny data
Section 4 — Full Code 
import numpy as np

# Input: [study_hours, sleep_hours]; Output: marks (percentage)
X = np.array(([2, 9], [1, 5], [3, 6]), dtype=float)
y = np.array(([92], [86], [89]), dtype=float)

# Normalize features and target to [0, 1] range
X = X / np.amax(X, axis=0)
y = y / 100

def sigmoid(x):
    """Activation function: squashes values to (0, 1)."""
    return 1 / (1 + np.exp(-x))

def derivatives_sigmoid(x):
    """Derivative of sigmoid for backpropagation."""
    return x * (1 - x)

# Hyperparameters
epoch = 7000          # Training iterations
lr = 0.1              # Learning rate
inputlayer_neurons = 2
hiddenlayer_neurons = 3
output_neurons = 1

# Initialize weights and biases randomly
wh = np.random.uniform(size=(inputlayer_neurons, hiddenlayer_neurons))
bh = np.random.uniform(size=(1, hiddenlayer_neurons))
wout = np.random.uniform(size=(hiddenlayer_neurons, output_neurons))
bout = np.random.uniform(size=(1, output_neurons))

for i in range(epoch):
    # --- Forward propagation ---
    hinp1 = np.dot(X, wh)
    hinp = hinp1 + bh
    hlayer_act = sigmoid(hinp)

    outinp1 = np.dot(hlayer_act, wout)
    outinp = outinp1 + bout
    output = sigmoid(outinp)

    # --- Backpropagation ---
    EO = y - output                              # Output error
    outgrad = derivatives_sigmoid(output)
    d_output = EO * outgrad

    EH = d_output.dot(wout.T)                    # Error at hidden layer
    hiddengrad = derivatives_sigmoid(hlayer_act)
    d_hiddenlayer = EH * hiddengrad

    # --- Weight & bias updates (gradient descent) ---
    wout += hlayer_act.T.dot(d_output) * lr
    bout += np.sum(d_output, axis=0, keepdims=True) * lr
    wh += X.T.dot(d_hiddenlayer) * lr
    bh += np.sum(d_hiddenlayer, axis=0, keepdims=True) * lr

    # Print progress every 1000 epochs
    if i % 1000 == 0:
        loss = np.mean(np.square(y - output))
        print(f"Epoch {i}/{epoch}, Loss: {loss}")
        print("Input:\n", X)
        print("Actual Output:\n", y)
        print("Predicted Output:\n", output)
        print("-" * 50)

Q1: What is an Artificial Neural Network?
Answer: A computational model inspired by biological neurons, consisting of interconnected layers that learn mappings from inputs to outputs through weight adjustments.
Q2: What is backpropagation?
Answer: An algorithm that computes gradients of the loss function with respect to each weight by applying the chain rule, propagating error backward from output to input layers.
Q3: Why normalize input data?
Answer: Normalization brings features to a similar scale, preventing large-valued features from dominating and helping gradient descent converge faster.
Q4: What is the purpose of the sigmoid function?
Answer: It introduces non-linearity and squashes output to (0,1), suitable for normalized targets. Without activation, the network is just linear regression.
Q5: What is MSE loss?
Answer: Mean Squared Error = average of (actual − predicted)². Penalizes large errors more heavily; standard loss for regression.
Q6: What is a learning rate?
Answer: A hyperparameter (0.1 here) controlling step size during weight updates. Too large → overshooting; too small → slow convergence.
Q7: What is an epoch?
Answer: One complete pass through the entire training dataset. This program runs 7000 epochs.
Q8: Difference between weights and biases?
Answer: Weights scale inputs; biases shift the activation threshold. Both are learnable parameters.
Q9: Why 3 hidden neurons?
Answer: Enough capacity to capture non-linearity for 2 inputs. More neurons = more capacity but risk overfitting on small data.
Q10: What is forward propagation?
Answer: Computing output by passing inputs through each layer: input → hidden (sigmoid) → output (sigmoid).
Q11: What is the chain rule in backprop?
Answer: If loss depends on output, output depends on weights, then ∂loss/∂weight = ∂loss/∂output × ∂output/∂weight.
Q12: What is vanishing gradient?
Answer: With sigmoid, derivatives near 0 for extreme inputs, causing gradients to shrink in deep networks, slowing learning.
Q13: Is this classification or regression?
Answer: Regression — output is continuous (marks), not discrete class labels.
Q14: What is the network architecture here?
Answer: 2-3-1: 2 inputs, 3 hidden neurons, 1 output neuron — a single hidden layer feed-forward network.
Q15: Why random weight initialization?
Answer: Breaks symmetry so neurons learn different features. Zero initialization would make all neurons identical.
Q16: What happens if learning rate is too high?
Answer: Weights oscillate or diverge; loss increases instead of decreasing.
Q17: What is gradient descent?
Answer: Iteratively update parameters in the direction opposite to the gradient of the loss to minimize error.
Q18: Can this network solve XOR?
Answer: No — a single hidden layer perceptron-like network with sigmoid may struggle; XOR needs non-linear separability with proper architecture.
Q19: What is the role of np.dot in ANN?
Answer: Performs matrix multiplication for weighted sum: each neuron's input = Σ(weight × input).
Q20: How to improve accuracy on this tiny dataset?
Answer: More data, tune learning rate/epochs, try different hidden sizes, or use sklearn MLPRegressor for comparison.

Perceptron for AND and OR Logic Gates

Section 1 — Aim

To design and train a single-layer Perceptron to implement AND and OR Boolean logic gates using the perceptron learning rule.

Section 2 — Problem Statement

Implement a Perceptron class with step activation, train it on AND gate truth table, then on OR gate truth table, and verify predictions on all 4 input combinations.

Section 3 — Theory

ML Concept

The Perceptron is the simplest neural network unit — a binary linear classifier. It learns a decision boundary w·x + b ≥ 0.

Why Perceptron?

AND and OR are linearly separable — a single line (hyperplane) can divide classes. Perceptron converges for linearly separable data.

Key Formulas

Output: y = step(w · x + b) = 1 if w·x+b ≥ 0, else 0
Update rule: w ← w + η · (t − y) · x
Bias update: b ← b + η · (t − y)

where η = learning rate, t = target, y = prediction.

Advantages

  • Simple, fast, easy to implement
  • Guaranteed convergence for linearly separable data
  • Foundation for understanding neural networks

Limitations

  • Cannot learn XOR (not linearly separable)
  • Only binary step output — not probabilistic
  • Single layer — no hidden features
Section 4 — Full Code 
import numpy as np

def step_function(x):
    """Returns 1 if x >= 0, else 0 (Heaviside step)."""
    return 1 if x >= 0 else 0

class Perceptron:
    def __init__(self, input_size, learning_rate=0.1):
        self.weights = np.zeros(input_size)   # Initialize weights to zero
        self.bias = 0
        self.learning_rate = learning_rate

    def predict(self, inputs):
        """Compute weighted sum + step activation."""
        summation = np.dot(inputs, self.weights) + self.bias
        return step_function(summation)

    def train(self, X, y, epochs=10):
        """Perceptron learning rule for each sample."""
        for epoch in range(epochs):
            for i in range(len(X)):
                prediction = self.predict(X[i])
                error = y[i] - prediction
                # Update weights and bias proportional to error
                self.weights += self.learning_rate * error * X[i]
                self.bias += self.learning_rate * error
            print(f"Epoch {epoch+1}: Weights: {self.weights}, Bias: {self.bias}")
        print(f"\nFinal Weights after training: {self.weights}")
        print(f"\nFinal bias after training: {self.bias}")

# AND gate truth table
X_and = np.array([[0, 0], [0, 1], [1, 0], [1, 1]])
y_and = np.array([0, 0, 0, 1])

# OR gate truth table
X_or = np.array([[0, 0], [0, 1], [1, 0], [1, 1]])
y_or = np.array([0, 1, 1, 1])

# Train AND perceptron
perceptron_and = Perceptron(input_size=2)
print("Training perceptron for AND gate")
perceptron_and.train(X_and, y_and)

# Train OR perceptron
perceptron_or = Perceptron(input_size=2)
print("Training perceptron for OR gate")
perceptron_or.train(X_or, y_or)

# Test both gates
print("\nTesting predictions")
test_inputs = np.array([[0, 0], [0, 1], [1, 0], [1, 1]])

print("\nPredictions for AND gate")
for test in test_inputs:
    print(f"Input: {test} -> Prediction: {perceptron_and.predict(test)}")

print("\nPredictions for OR gate")
for test in test_inputs:
    print(f"Input: {test} -> Prediction: {perceptron_or.predict(test)}")

K-Means Clustering on Iris Dataset

Section 1 — Aim

To perform K-Means clustering on the Iris dataset, visualize real vs predicted clusters, and display cluster centroids.

Section 2 — Problem Statement

Load Iris dataset, apply KMeans with k=3 clusters on 4 features, compare actual species labels with K-Means assigned labels using scatter plots of petal length vs petal width.

Section 3 — Theory

ML Concept

Unsupervised learning — K-Means partitions data into k clusters by minimizing within-cluster variance.

Algorithm Steps

  1. Initialize k centroids randomly
  2. Assign each point to nearest centroid
  3. Recompute centroids as mean of assigned points
  4. Repeat until convergence

Key Formulas

Objective: J = Σ Σ ||xi − μj||²
Centroid update: μj = (1/|Cj|) Σ xi for xi ∈ Cj

Advantages

  • Simple, fast, scalable
  • Works well on spherical clusters

Limitations

  • Must specify k in advance
  • Sensitive to initialization and outliers
  • Assumes equal-variance spherical clusters

Section 3 — Theory

ML Concept

K-Modes is the categorical counterpart of K-Means. It uses mode (most frequent value) instead of mean for centroids and dissimilarity based on matching categories.

Why K-Modes?

K-Means uses Euclidean distance on numeric data — meaningless for strings like 'Red', 'Circle'. K-Modes handles nominal categorical attributes directly.

Key Formulas

Dissimilarity: d(X, Y) = Σ δ(xi, yi) where δ = 0 if match, 1 if not
Mode: most frequent category value in cluster for each attribute

Advantages

  • Designed for categorical/nominal data
  • Simple extension of K-Means concept
  • Interpretable cluster centroids as category modes

Limitations

  • Requires kmodes library (not in sklearn)
  • Sensitive to initial cluster centers
  • Only for categorical — mixed data needs K-Prototypes

Section 1 — Aim

To perform agglomerative hierarchical clustering on Iris data using Ward's linkage with dendrogram visualization, and implement divisive (top-down) clustering recursively.

Section 2 — Problem Statement

Section 3 — Theory

ML Concept

Hierarchical clustering builds a tree (dendrogram) of clusters. Agglomerative (bottom-up) merges closest pairs; divisive (top-down) splits clusters recursively.

Ward's Linkage

Minimizes increase in total within-cluster variance when merging two clusters — tends to produce compact, equal-sized clusters.

Key Formulas

Ward distance: Δ(A,B) = increase in SSE when merging clusters A and B

Advantages

  • No need to specify k upfront — cut dendrogram at desired level
  • Dendrogram provides rich visualization

Limitations

  • O(n³) time — slow on large datasets
  • Irreversible merges in agglomerative approach

def divisive_clustering(X, n_clusters=3):
    """Top-down: recursively split into 2 until n_clusters reached."""
    if n_clusters <= 1:
        return [X]
    model = AgglomerativeClustering(n_clusters=2)
    labels = model.fit_predict(X)
    cluster_1 = X[labels == 0]
    cluster_2 = X[labels == 1]
    result = []
    if len(cluster_1) > 1:
        result += divisive_clustering(cluster_1, n_clusters=n_clusters // 2)
    if len(cluster_2) > 1:
        result += divisive_clustering(cluster_2, n_clusters=n_clusters // 2)
    return result

divisive_result = divisive_clustering(X, n_clusters=3)
plt.figure(figsize=(10, 7))
colors = ['blue', 'green', 'red', 'orange', 'purple']
for idx, cluster in enumerate(divisive_result):
    plt.scatter(cluster[:, 0], cluster[:, 1], label=f'Cluster {idx + 1}', color=colors[idx % len(colors)])
plt.title("Divisive Hierarchical Clustering (Top-down Approach)")
plt.xlabel("Sepal Length")
plt.ylabel("Sepal Width")
plt.legend()
plt.show()

Section 3 — Theory

ML Concept

SVM finds optimal hyperplane maximizing margin between classes. Support vectors are points closest to the boundary.

Decision function: f(x) = w·x + b; class = sign(f(x))
Objective: minimize ||w||² subject to yᵢ(w·xᵢ + b) ≥ 1

Parameter C

C controls trade-off: large C = hard margin (less tolerance for misclassification); small C = soft margin.

Advantages

  • Effective in high dimensions
  • Memory efficient (uses support vectors only)

Limitations

  • Slow on very large datasets
  • Linear kernel limited for non-linear boundaries

Q-Learning in Gridworld Environment

Section 1 — Aim

To implement Q-learning reinforcement learning algorithm in a 3×3 Gridworld and visualize the learned optimal policy.

Section 2 — Problem Statement

Create Gridworld with start (0,0) and goal (2,2), train Q-table for 1000 episodes using epsilon-greedy exploration, update Q-values with Bellman equation, and display optimal action arrows.

Section 3 — Theory

ML Concept

Reinforcement Learning — agent learns by interacting with environment, receiving rewards, maximizing cumulative reward.

Q-update: Q(s,a) ← Q(s,a) + α[r + γ·max Q(s',a') − Q(s,a)]

Program uses simplified update: Q(s,a) ← r + γ·max Q(s',a')

Parameters

  • α=0.1 learning rate
  • γ=0.9 discount factor
  • ε=0.1 exploration rate

Advantages

  • Model-free — no environment dynamics needed
  • Converges to optimal policy with sufficient exploration

Limitations

  • Tabular Q-table doesn't scale to large state spaces
  • Simplified update in code omits learning rate on TD error
Section 4 — Full Code 
import numpy as np
import matplotlib.pyplot as plt

class Gridworld:
    def __init__(self):
        self.size = (3, 3)
        self.goal = (2, 2)
        self.state = (0, 0)

    def reset(self):
        self.state = (0, 0)
        return self.state

    def step(self, action):
        # Actions: 0=up, 1=right, 2=down, 3=left
        actions = [(0, -1), (1, 0), (0, 1), (-1, 0)]
        next_state = (self.state[0] + actions[action][0],
                      self.state[1] + actions[action][1])
        if 0 <= next_state[0] < 3 and 0 <= next_state[1] < 3:
            self.state = next_state
        return self.state, 0 if self.state == self.goal else -1

env = Gridworld()
q_table = np.zeros((3, 3, 4))  # state_x, state_y, action
alpha, gamma, epsilon = 0.1, 0.9, 0.1
episodes = 1000

for _ in range(episodes):
    state = env.reset()
    done = False
    while not done:
        # Epsilon-greedy action selection
        if np.random.rand() < epsilon:
            action = np.random.choice(4)
        else:
            action = np.argmax(q_table[state[0], state[1]])
        next_state, reward = env.step(action)
        q_table[state[0], state[1], action] = (
            reward + gamma * np.max(q_table[next_state[0], next_state[1]])
        )
        state = next_state
        if state == env.goal:
            done = True

# Visualize optimal policy
fig, ax = plt.subplots(figsize=(5, 5))
ax.set_ylim(3, 0)
ax.set_xlim(0, 3)
actions = ['↑', '→', '↓', '←']
for y in range(3):
    for x in range(3):
        action = np.argmax(q_table[x, y])
        ax.text(x + 0.5, y + 0.5, actions[action], ha='center', va='center', fontsize=12)
ax.text(2.5, 2.5, "G", ha='center', va='center', fontsize=12, color="green")
plt.title("Q-learning Optimal Policy")
plt.show()

Genetic Algorithm for Function Optimization

Section 1 — Aim

To implement a Genetic Algorithm to maximize f(x) = x² over the interval [-5, 5] using selection, crossover, and mutation operators.

Section 2 — Problem Statement

Initialize population of 20 random values, run 100 generations with tournament selection, averaging crossover, and random mutation, track best fitness, and plot fitness progression.

Section 3 — Theory

ML Concept

Genetic Algorithms (GA) are evolutionary optimization inspired by natural selection — populations evolve toward optimal solutions.

GA Steps

  1. Initialize population
  2. Evaluate fitness
  3. Select parents (tournament)
  4. Crossover (recombine)
  5. Mutate (random changes)
  6. Repeat for generations
Fitness: f(x) = x² (maximize on [-5, 5])

Advantages

  • Global search — avoids local minima
  • No gradient required

Limitations

  • No guarantee of global optimum
  • Many hyperparameters to tune
Section 4 — Full Code 
import numpy as np
import random
import matplotlib.pyplot as plt

def fitness(x):
    return x ** 2

def initialize_population(pop_size, lower, upper):
    return np.random.uniform(lower, upper, pop_size)

def select_parents(population, fitness_values):
    parents = []
    for _ in range(len(population) // 2):
        tournament = random.sample(list(zip(population, fitness_values)), 3)
        parents.append(max(tournament, key=lambda x: x[1])[0])
    return np.array(parents)

def crossover(parents):
    offspring = []
    for i in range(0, len(parents), 2):
        if i + 1 < len(parents):
            child = (parents[i] + parents[i + 1]) / 2
            offspring.append(child)
    return np.array(offspring)

def mutate(population, mutation_rate, lower, upper):
    for i in range(len(population)):
        if random.random() < mutation_rate:
            population[i] += np.random.uniform(-0.5, 0.5)
            population[i] = np.clip(population[i], lower, upper)
    return population

def genetic_algorithm(pop_size, generations, lower, upper, mutation_rate):
    population = initialize_population(pop_size, lower, upper)
    fitness_history = []
    for generation in range(generations):
        fitness_values = np.array([fitness(x) for x in population])
        best_fitness = np.max(fitness_values)
        fitness_history.append(best_fitness)
        if generation < 3 or generation == 99:
            print(f"Generation {generation + 1} - Best Fitness: {best_fitness}")
        parents = select_parents(population, fitness_values)
        offspring = crossover(parents)
        while len(offspring) < len(population):
            offspring = np.append(offspring, parents[random.randint(0, len(parents) - 1)])
        population = mutate(offspring, mutation_rate, lower, upper)
    best_solution = population[np.argmax([fitness(x) for x in population])]
    return best_solution, fitness_history

best_solution, fitness_history = genetic_algorithm(20, 100, -5, 5, 0.1)
print(f"Best solution: {best_solution}")
print(f"Best fitness (f(x) = x^2): {fitness(best_solution)}")
plt.plot(fitness_history)
plt.xlabel('Generation'); plt.ylabel('Best Fitness Value')
plt.title('Fitness Progression Over Generations')
plt.show()

  4. Test on all examples

  • May overgeneralize or undergeneralize

import numpy as np

dataset = [
    {"Age": 22, "Income": 30000, "Buy Product": "No"},
    {"Age": 25, "Income": 50000, "Buy Product": "Yes"},
    {"Age": 45, "Income": 80000, "Buy Product": "Yes"},
    {"Age": 35, "Income": 40000, "Buy Product": "No"},
]

def satisfies_condition(example, condition):
    return condition(example)

def ebl(dataset):
    explanation = None
    for example in dataset:
        if example["Buy Product"] == "Yes":
            explanation = lambda x: (25 <= x["Age"] <= 45) and (x["Income"] > 40000)
            break
    return explanation

def test_explanation(dataset, explanation):
    tp = fp = tn = fn = 0
    predictions = []
    for example in dataset:
        predicted = "Yes" if satisfies_condition(example, explanation) else "No"
        predictions.append(predicted)
        if predicted == example["Buy Product"]:
            if predicted == "Yes": tp += 1
            else: tn += 1
        else:
            if predicted == "Yes": fp += 1
            else: fn += 1
    accuracy = correct = tp + tn
    accuracy = accuracy / len(dataset) * 100
    return accuracy, predictions, tp, fp, tn, fn

def run():
    explanation = ebl(dataset)
    print("Learned Rule: If age 25-45 and income > 40000, they will buy.")
    accuracy, predictions, tp, fp, tn, fn = test_explanation(dataset, explanation)
    print(f"Explanation-based learning accuracy: {accuracy}%")
    for i, example in enumerate(dataset):
        print(f"Example {i+1}: Predicted = {predictions[i]}, Actual = {example['Buy Product']}")
    print(f"True Positives: {tp}, False Positives: {fp}")

  • "Can K-Means handle categorical data?" → No, use K-Modes.
  • "Does DBSCAN need k?" → No, but needs eps and min_samples.
  • "Is Q-learning supervised?" → No, it's reinforcement learning.
